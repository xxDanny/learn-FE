    使用代码切换路由

    # 使用代码切换路由

    除了在模板中利用路由链接组件来触发路由请求，也可以调用路由器实例的以下方法，实现程序化导航

    # push()

    调用路由器实例的 push() 方法来请求一个新的路由，并在成功匹配后向历史访问栈压入当前路径信息：

    push() 方法的参数可以是一个目标路径，例如 this.$router.push('/blog') ，也可以 是一个定位对象，例如： this.$router.push({name:'blog'}) 。

    # replace()

    replace() 方法也用来请求一个新的路由，不过它和 push() 的区别在于，在成功匹配路由后， replace() 方法将使用这个新的路由信息替换历史访问栈的当前位置信息：

    和 push 一样， replace() 方法的参数可以是一个目标路径，例如 this.$router.replace('/blog') ， 也可以是一个指向命名路由的对象，例如： this.$router.replace({name:'blog', params:{idx:'1'}}) 。

    # go()

    go() 方法根据指定的偏移量在访问历史栈中重新定位：

    偏移量是一个整数，正整数表示向前（栈增长的方向）移动，负整数表示向后移动。例如 this.$router.go(-1) 表示从当前位置退回前一个路由， this.$router.go(1) 表示从当前位置前进一个路由。

    容易理解，路由器提供的另外两个方法 forward() 和 back() ，分别是对 go(1) 和 go(-1) 的简单封装。

    go() 方法只是改变历史访问栈的当前访问定位（影响 replace() 方法的结果），并不会 改变栈的内容。

    # 查看历史访问栈

    VueRouter 内置了三种历史访问栈的实现： abstract 、 hash 和 history ，默认情况下启用的是 hash 访问栈。遗憾的是，后两种访问栈都是基于浏览器的 history 开发接口 实现，而 history 没有提供查看栈内容的接口 😦

    好在三种访问栈的运行机制在 VueRouter 层面是一样的，我们可以利用 VueRouter 的实例化 配置项 mode 来启用 abstract 访问栈，通过查看其访问栈（通过 $router.history.stack 访问） 的内容，来理解路由器的这些方法对历史访问栈的操作。

    例如：

    const router = new VueRouter({
  routes: [...],
  mode: 'abstract' //启用 abstract 访问栈
})
new Vue({
  router: router,
  template: `<div>{{$router.history.stack}}</div>`
})
