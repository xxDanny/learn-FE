    Jest中的钩子函数

    # Jest中的钩子函数

    钩子函数是指在 Jest运行过程中的某一时期是自动调用的函数,这个概念和 vue 以及 react 中的生命周期函数类似

    # 编写一个计数器对象

    新建 Counter.js 文件，用 ES6 语法编写一个计数器对象 Counter。包含两个方法 addOne，minusOne方法

    export default class Counter {
    constructor() {
        this.number = 0;
    }
    addOne() {
        this.number += 1
    }
    minusOne() {
        this.number -= 1;
    }
}

    # 编写测试 addOne 和 minusOne 方法的测试用例

    新建 Counter.test.js 文件，编写测试 addOne 和 minusOne 方法的测试用例

    import Count  from './Counter';

let counter = new Count();

test('测试 Counter 中的 addOne 方法',() => {
    counter.addOne()
    expect(counter.number).toBe(1)
})

test('测试 Counter 中的 minusOne 方法',() => {
    counter.minusOne()
    expect(counter.number).toBe(0)
})

    ❗️但是这样编写会有一个问题是，每个测试用例都共享一个 counter 实例。会高度耦合，我们怎样实现每个测试用例都有自己的 counter 实例呢？

    💡我们可以借助 beforeEach 生命周期函数实现 Jest中主要有四个核心生命周期函数：beforeAll,afterAll,beforeEach,afterEach

    # beforeAll

    beforeAll 钩子函数是在所有测试用例执行之前执行。你可以在beforeAll 中做一些准备工作。 但是在 beforeAll 中初始化一个 counter 实例是有问题的，因为每个测试都共用一个 counter 实例。

    # beforeEach

    beforeEach 是在每个测试用例执行之前执行。所以我们在 beforeEach 中生成 counter 实例。这样每个测试用例都会有一个独立的 counter 实例并且互不干扰。

    改写测试用例如下

    // 🗂 Counter.test.js
import Counter  from './Counter';

let counter;
// 利用 beforeEach 钩子函数在测试用例开始之前创建一个 counter 实例
beforeEach(() => {
    counter = new Counter();
})

test('测试 Counter 中的 addOne 方法',() => {
    counter.addOne()
    expect(counter.number).toBe(1)
})

test('测试 Counter 中的 minusOne 方法',() => {
    counter.minusOne()
    expect(counter.number).toBe(-1)
})

    但是此时如果我们想要把功能进行分组，把测试加法的功能函数算作一组，测试减法的一组，我们应该怎么实现呢？

    测试我们可以使用 describle 函数,这样测试代码结构层次会更加清晰

    // 🗂 Counter.test.js
describe('测试 Counter 中加法相关功能', () => {
    test('测试 Counter 中的 addOne 方法',() => {
        counter.addOne()
        expect(counter.number).toBe(1)
    })
    test('测试 Counter 中的 addTwo 方法',() => {
        counter.addTwo()
        expect(counter.number).toBe(2)
    })

    })

    test('测试 Counter 中的 minusTwo 方法',() => {
        counter.minusTwo()
        expect(counter.number).toBe(-2)

    # 钩子函数的作用域

    每一个 describe 下都可以拥有自己的钩子函数，使用的钩子函数对自己的子 describe 的测试用例也适用。 说明钩子函数是有作用域的，而且在 describe 的范围内，至于执行顺序是由外到内。

    # 测试单个用例

    当我们测试用例代码越写越多时，我们想要只测试某个用例，忽略其他测试用例时，可以运用 .only 修饰符，它只会执行加上这个标识的测试用例，跳过其他的测试用例

    test.only('测试 Counter 中的 minusTwo 方法',() => {
        counter.minusTwo()
        expect(counter.number).toBe(-2)
})

    # describe内的语句执行顺序

    如果一段代码写在 describe 里，没有写在 describe 里面的生命周期函数里，会被优先执行

    describe('测试 Counter 对象',() => {
    beforeEach(() => {
        console.log('最外部的 beforeEach')
        counter = new Counter();
    })

    describe('测试 Counter 中加法相关功能', () => {
        beforeEach(() => {
            console.log('测试加法的 beforeEach')
        })
        console.log('测试加法的 describle')
        test.only('测试 Counter 中的 addOne 方法',() => {
            counter.addOne()
            expect(counter.number).toBe(1)
        })
        test('测试 Counter 中的 addTwo 方法',() => {
            // do something
        })
    })

    describe('测试 Counter 中减法相关功能', () => {
       // do something...
    })

})
