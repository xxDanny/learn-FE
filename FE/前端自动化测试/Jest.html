    Jest 测试框架入门

    # Jest 测试框架入门

    # 使用 Jest 修改自动化测试样例

    1. 安装 node 环境
    2. 按住 ctrl + ～ 打开终端面板
    3. 输入 node -v 查看 node 版本验证 node 是否安装成功
    4. 要想安装 Jest 框架，必须要运行 npm init ，初始化文件夹为 npm 包

    Q:npm init 有什么作用？

    1. 输入 npm install jest@24.8.0 -D 安装 Jest 24.8.0 版本

    Q: -D有什么作用？ A: 将框架保存在开发环境 dev 中，因为上线不需要使用 Jest 框架了。

    1. 编写自动化测试脚本
    2. 进入 package.json 文件 ,改写 scripts。运行 npm run test 就会执行 jest
      "scripts": {
    "test": "jest"
  },
    1
    2
    3

    Q: jest 命令会执行什么？ A: 会找到以 test.js 结尾的文件并执行

    Jest 主要完成单元测试和继承测试，所有测试的所有都是模块，必须改造成模块，才能进行自动化测试。但是在使用 vue 或者 react 进行开发时，已经是按照模块化进行开发了。

    # Jest 的简单配置

    🔗Jest 的简单配置 | github (opens new window)

    • Q：npm install作用？
    • A：根据 paackage.json 安装依赖

    # 开始配置 Jest

    1. 输入 npx jest --init 开始配置 jest，成功配置之后，目录下出现 jest.congif.js 文件
    2. jest.congif.js 文件是对 jest 进行各种配置，例如：
        clearMocks: true
    1

    表示是否需要 jest 自动清除 mock 数据

        coverageDirectory: 'coverage',
    1

    表示 jest 将自动生成测试覆盖率报告并且存储在 coverage 文件夹下面

    # 测试覆盖率报告

    运行 npx jest --coverage，就可以在控制台生成测试报告，并且生成coverage 文件以及相关覆盖率报告

    上图表示只覆盖了 33% 的函数测试

    在浏览器打开 coverage -> lcov-report -> index.html ，可以看到 jest 自动生成美观的测试覆盖率报告,如下图

    但是每次都需要输入 npx jest --coverage 有点麻烦，我们可以做出一点改进。进入 package.json , 在 scripts 字段下添加

    "coverage" : "jest --covergae"
    1

    以后只要输入 npm run coverage 就可以啦😄

    # 如何测试 ES Module 代码

    当我们使用 vue 或者 react 编写代码时，常常使用的 ES module 模块化语法，那又应该如何编写呢？

    因为 Jest 只认识 commonjs 的模块语法。不认识 ES module 模块语法，因此运行 npm run test 会报错

    其实很好解决，我们只要使用 babel 将 Es module 模块语法转为 commonjs 模块语法

    1. 首先安装 babel ,输入如下命令：
    npm install @babel/core@7.4.5  @babel/preset-env@7.4.5 -D
    1
    1. 配置 babel， 在根目录创建 .babelrc 文件
    {
    "presets": [
        ["@babel/preset-env",{
            "targets": {
                "node": "current"
            }
        }]
    ]
}

    现在运行 npm run test 就不会报错了。

    # toBe 匹配器

    首先什么是匹配器？查看 matchers.test.js 中写的一个测试用例。

    test('测试加法 3 + 7', () => {
    expect(add(3, 7)).toBe(10);
})
    1
    2
    3

    这里的 toBe 方法就是一个匹配器，类似于 JavaScript 中的 object.is方法 或者 ====

    // matchers.test.js
// ❌测试失败
test('对象 a 是否等于 { one : 1 }', () => {
    const a = { one : 1 }
    expect(a).toBe({ one : 1 });
})
    1
    2
    3
    4
    5
    6

    运行会报错，因为 对象 a和 对象直接量 { one : 1 } 的引用地址不同，即便内容相同使用 toBe 匹配器去匹配依然会失败

    # toEqual 匹配器

    因此当你编写测试用例只想测试对象内容相等时，就可以使用 toEqual 匹配器。

    # Jest 自动监听运行

    每次我们改代码，都要运行 npm run test 有些麻烦。这里我们可以再做一点改进 进入 package.json 文件,在 jest 后添加 --watchAll。

     "scripts": {
    "test": "jest --watchAll",
  },
    1
    2
    3

    --watchAll 的作用是让 jest 监听所有测试用例的变化，当我们改变测试用例时会自动运行 npm run test，从而提高编写测试用例的效率

    # Jest 命令行工具使用

    首先我们来讲一个 vscode 小技巧，在终端切换目录时，希望 vscode 打开某一目录

    1. 打开 vscode ，按住 command + shift + p, 这是 vscode 会弹出一个命令窗口

    1. 搜索 install code command,点击给出的指令

    1. vscode 会给出如下提示,code 指令被成功安装在系统 Path 目录中

    我们来看如何使用 code 值令

    1. 从终端进入 jest 命令行工具 文件夹,如果我们想用 vscode 打开这个文件夹，只需要在终端输入 code ./,就可自动唤醒 vscode 打开jest 命令行工具 文件夹

    # o 模式

    o 模式 只有代码改动才会执行被改变的文件夹 Press o to only run tests related to changed files. 代表按 o 只执行修改的文件。此时你按下 o 但是会报错。那是因为文件没有绑定git，所以无法知道那些文件修改过了，所以我们还需要安装 git 环境

    1. 打开 git 官网，下载安装 git
    2. 在终端输入 git init,初始化目录为空的 git 仓库

    另外一种方法进入 o 模式，进入 package.json 文件，将 "test": "jest --watchAll", 改为 "test": "jest --watch",

    # f 模式

    f 模式 只会运行失败的测试用例

    # a 模式

    a 模式会运行所有测试用例

    # t 模式

    根据测试用例名称，只运行想要测试的测试用例

    # q 模式

    按下 q 会推出对测试用例的监听
