    生命周期和模板引用

    # 生命周期和模板引用

    # vue 生命周期函数

    1. beforeCreate ：创建一个空白 vue 实例。data，method 尚未初始化，不可使用
    2. created：vue 实例创建完毕，完成响应式绑定，尚未开始渲染。data，method 初始化完成，可调用。这里可以处理 js 相关的事务，但是不能处理 dom。
    3. beforeMount：编译模版，调用 render 生成 vdom，还没有渲染 dom
    4. mounted：完成的 dom 渲染，组件创建完成。由 创建 阶段进入运行阶段
    5. beforeUpdate：data 发生变化之后，准备更新 dom（尚未更新 dom）
    6. updated：更新了 dom
    7. beforeUnmount：组件进入销毁阶段，可以移除一些全局事件，自定义时间
    8. unmounted: 组件销毁了

    # vue 什么时候操作 dom 合适

    mounted 和 updated 不能保证子组件全部挂载完成 使用 $nextTick 渲染 DOM

    mounted() {
  this.$nextTick(function() {
    // 整个视图被渲染完成后才厚执行
  })
}

    2. 使用 hooks 代替了 mounted（）

    # 浏览器和 nodejs 事件循环区别

    js 是单线程的无论浏览器还是 nodejs

    JS 执行和 DOM 渲染是共用一个线程：异步就是解决方案

    微任务是在下一轮 DOM 渲染之前执行，宏任务是在下一轮 DOM 渲染之后就执行

    微任务： this.nextTick()，promise

    同步代码执行，遇到异步函数，浏览器会把异步回调函数放在回调函数队列中，同步代码结束之后，通过事件循环机制访问回调函数队列，执行队列重的任务。执行完之后，event loop 继续监听有没有新的任务进入任务队列中

    nodejs 也适用 es 语法，也有红任务和微任务，但是分不同类型，不同优先级

    宏任务；

    1. setTimeout
    2. I/O 回调函数 - 处理网络，流，TCP 回调错误
    3. Idle，prepare - 闲置状态（nodejs 使用）
    4. poll 轮询 - 执行 poll 中 I/O 队列
    5. check - 存储 setImmediate 回调
    6. close callbacks - 关闭回调。如 socket.on（‘close’）

    微任务 process.nextTick() promise，async/await

    # 虚拟 DOM

    用 js 节点 模拟 dom 节点的数据，最开始是由 react 推出的

    vue/react 价值：1. 组件化 2. 数据视图分离，数据驱动分离

    之前需要用 js 手动操作 dom，很多繁琐的代码。 model 操作 view。js 只需要操作数据的变化就可以了。我们就你可以只关注业务数据，不用再关心 Dom 变化

    如何实现数据驱动视图：VDOM

    业务发生变化之后，会触发模版重新生成新的 Vdom，利用 diff 算法然后新旧 Vdom 进行对比，找出需要更新的点最后去更新 dom。

    简化开发逻辑，简化了代码，更有利于沟通，开发，重构。

    目前为止，Vue 为我们处理了所有的 DOM 更新，这要归功于响应性和声明式渲染。然而，有时我们也会不可避免地需要手动操作 DOM。

    这时我们需要使用模板引用——也就是指向模板中一个 DOM 元素的 ref。我们需要通过这个特殊的 ref attribute 来实现模板引用：

    <p ref="p">hello</p>
    1

    此元素将作为 this.$refs.p 暴露在 this.$refs 上。然而，你只能在组件挂载之后访问它。

    要在挂载之后执行代码，我们可以使用 mounted 选项：

    export default {
  mounted() {
    // 此时组件已经挂载。
  }
}
