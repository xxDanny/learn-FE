    # 发布订阅模式

    目的：让各个模块之剪没有紧密的联系

    现在处理办法是属性更新的时候，调用 mountComponent

    mountComponent 更新了什么呢？以组件为单位进行判断，以节点为单位进行局部更新

    复杂的页面是由很多组件构成的，每一个属性要更新都要调用更新的方法。每个组件都会有自己的渲染方法

    目标：如果修改了属性->只更新这个属性对应的页面的 DOM，这样就一定不能把更新代码写死

    例子：预售，没有现货，你告诉老板这个衣服到了就发个消息。

    老板就是发布者 我就是订阅者

    用代码来描述：

    • 老板提供一个预售衣服订单列表（数组）
    • 我订阅我需要的衣服（老板记录下睡订了哪件衣服在数组中存储某项描述）
    • 等待。
    • 当衣服到货，老板查看衣服订单列表，挨个打电话（遍历数组，取出数组元素来使用）

    实际上就是事件模型：

    1. 有一个 event 对象
    2. on，off，emit 方法

    我们要实现事件模型，思考怎么用？

    1. event 是一个全局对象
    2. event.on('事件名'，处理函数） - 订阅事件
      • 事件可以连续订阅
      • 可以移除
    3. 写别的代码
    4. event.emit('事件名'，参数） 先前注册的事件处理函数就用调用
    event.on('click', fn)
event.emit('click')

    // 全局对象 event
var event = (function() {
  var eventObjs = {} // 事件队列
  return {
    /**注册事件，可以连续注册，可以注册多个时间**/
    on: function(type, handler) {
      if (!eventObjs[type]) {
        eventObjs[type] = [];
        eventObjs[type].push(handler)
      }
    },
    /**移除事件
     * 如果没有参数，移除所有事件。
     * 如果只带有事件名，移除事件名下所有事件
     * 如果有两个参数，移除某一个事件名下具体的处理函数
     **/
    off: function(type, handler) {
      if (arguments.length == 0) {
        eventObjs = {}
      } else if (arguments.length == 1) {
        eventObjs[type] = [];

      } else if (arguments.length == 2) {
        let _events = eventObjs[type];
        //倒着循环，数组序号不会受影响
        for (let i = _events.length - 1; i >= 0; i--) {
          if (_events[i] === handler) {
            _events.splice(i, 1)

          }
        }

      }
    },

    emit: function(type) {
      let arg = Array.prototype.slice.call(arguments, 1); // 获取除了 type 之后所有参数
      let _events = eventObjs[type];
      for (let i = _events.length - 1; i >= 0; i--) {
        _events[i].apply(null, args)

      }
    }
  }
}

    VUE 模型如何实现发布订阅模式？

    如何触发 DOM 更新？

    有一个全局的容器存储 watcher，每个组件都有一个 update 方法，对应一套 watcher

    watcher 就是更新方法，一一对应响应式的数据

    这些响应式的数据读取的时候，调用 depend 方法，将对应 watcher 存入全局 watcher 中的 那什么时候会读取数据呢？全局渲染的时候，虚拟 DOM 生成的时候

    设置数据的时候，会调用 notify 方法，将全局所有 watcher 一一触发

    # watcher 和 dep

    每一个 watcher 用来描述渲染行为或者计算行为。例如 vue 推荐使用计算属性代替复杂的插值表达式，计算属性会伴随其使用的属性变化而变化 (watcher 完成）。

    依赖收集与派发中心怎么运行起来的？

    在访问的时候就会收集，在修改的时候就会更新

    所谓的依赖收集就是告诉 watcher 什么属性被访问了，那么在这个 watcher 渲染页面时，就会将这些收集的属性进行更新

    Watcher 实现：

    1. 只考虑修改后刷新（响应式）
    2. 再考虑依赖收集（优化）

    在 VUE 中提供一个 watcher 构造函数

    • get()：完成计算功能或执行处理函数
    • update(): 该方法会触发内部的 run 方法
    • run()：运行，用来判断内部是异步运行还是同步运行，这个方法最终会调用内部的 get 方法
    • cleanupDep：简单理解为清除队列

    我们的页面渲染是上面哪个方法执行的？ - get 方法

    我们的 watcher 实例有一个属性 vm 表示当前 vue 实例

   *

    this.vm = vm;
    this.getter = expOrfn;

  get() {

    this.getter.call(this.vm, this.vm)
  }

  run() {
    this.get()
  }

  update() {
    this.run()
  }

  cleanupDep() {

  }

}

    Q：如何将属性与当前 watcher 联系起来？

    • 在全局准备一个 (watcher 栈，简单理解为 watcher‘数组’，把一个操作中需要使用的 watcher 都存储起来）
    • 在 watcher 调用 get 方法的时候，将当前 watcher 放到全局，在 get 执行结束之后，将全局 watcher 移除
      • 提供 pushTarget，popTarget
    • 在每一个属性中都有一个 Dep 对象
      • 在属性响应式化的时候创建的，每一个属性都会有一个 DEP 对 象

    我们在访问对象属性的时候 ，我们的渲染 watcher 就在全局 将属性与 watcher 相关联就是将当前渲染的 watcher 存储到属性相关的 Dep 中

    同时，将 dep 也存储到全局 watcher 中（互相引用的关系）

    • 属性引用了当前的渲染 watcher，属性知道谁渲染它
    • 当前渲染 watcher 引用了访问的属 性（Dep），当前的 watcher 知道渲染了什么属性

    我们的 Dep 有一个方法 notify() 其内部就是将 Dep 中 subs 取出来，依次调用 update 方法

    subs 里面存的是要渲染什么属性的 watcher
