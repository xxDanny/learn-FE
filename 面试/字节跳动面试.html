    # 字节跳动面试

    某互联网知名企业发展迅猛，过程中遇到五个问题，分别大概是

    1. 人员方面（老员工工资高作为底，新员工工资低价值高，人才流行性大留不住）
    2. 职能方面 （职能部门交叉，职责分工不明确，效率低下）
    3. 管理层方面（中高层领导思维固化，缺乏创新意识）
    4. 战略方面 （公司总部与分部战略规划不明确）
    5. 资金方面（公司融资困难，资金链出现问题）

    问题：排列出你认为的五个问题的优先级，并就你列出的前两项提出具体的解决方案。

    1. 人员方面：主管离职
    2. 职能方面 （职能部门交叉，职责分工不明确，效率低下）
    3. 小张被分配到佳绩项目，与小 A 有矛盾冲突
    4. 小 B 是专家，但是不服从管理
    5. 老员工安于现状，不主动，只听从上级安排的任务。

    问题：排列出你认为的五个问题的优先级，并就你列出的前三项提出具体的解决方案。

    面试前准备：

    • pc 或者 ipad 下载好「飞书」，个人不建议用手机面试（原因下文会说）。
    • 纸，笔：方便题目简要记录和大家提出的观点进行整理归纳。

    # 一分钟自我介绍

    首先是 HR 点名答到，然后就开始一个简短的一分钟自我介绍主要包括下面三个方面

    • 个人基本信息（姓名+学校+专业）
    • 实习经历
    • 应聘岗位的理由

    个人认为群面自我介绍简要说一下自己的特点，以及和这个岗位匹配饿能力就行了，不用太详细地列举自己实习的丰功伟绩，获得了多少奖学金得了多少证书之类的，因为这个自我介绍主要的为了 HR 和小组成员认识你，和一对一的单面还是区别的。说的太华丽，也有利有弊，会让人对你期待很高，如果后面表现差强人意反而大打折扣。

    因为我昨晚才看见面试邀请，今早匆匆起来，自我介绍也是即兴的，说话都磕磕巴巴。当时已经感觉大事不妙，产生想要直接挂掉面试链接，开始摆烂的想法。但是后面还是硬逼着自己坚持下来，就当自己是来玩的，体验一下群面的流程和氛围，也是一次难能可贵的经历，才能有所成长。其实自我介绍的时候就有人心理成熟不住，退出了面试，至少我扛住了自我介绍也算是一个突破吧（社恐人士真的太害怕群面了😭）

    # 无领导小组讨论

    自我介绍过后，就开始下一个阶段也就是 45 分钟的无领导小组讨论。

    这里又有一个小插曲，大家一定要提前下在电脑上下载好飞书，我的电脑不知道除了什么事故，一直安装不上飞书，后来我就换的手机进行线上面试的。结果导致读题目的时候，字体太小，根据看不清。

    由于而且我没有群面经验，我以为题目会一直在屏幕上面，结果是五分钟读题过后，题目就消失了，画面切换到小组视频画面。当时我一整个大崩溃。所以大家一定要抓紧时间读题，简要记下题目重要信息，理解好题干。

    ::: tips 题目分享

    :::

    大家读完题后，大家可能是都还在各自梳理自己的想法，就陷入数分钟的沉默。首先是一个声音很好听的女生破冰，梳理了一遍我们需要解决的问题，让团能够明确我们共同需要解决的问题。个人觉得这样的梳理是很有必要的，像我这种第一次参加群面的菜鸡在读题目的时候，因为太紧张和手机屏幕字太小，一直读不进去，导致讨论的时候，我题目都没理解清楚，GG ，当时就希望有一个天使 leader 可以重新梳理一下题目要点。然后有一个在特斯拉实习过的男生开始控场，滔滔不绝发表自己的见解，虽然说的略显啰嗦，但是我还是羡慕。内心 os 自己什么时候能这么多话，能大胆的发言，

    其实相比自我介绍，这个部分社恐人士其实相对可以轻松一点，我当时已经处于摆烂心态，就果断选择当一个参与者，听听大家的想法。你不用担心场面会一度尴尬。因为总会有人想表现自己，一定有人说话的，甚至会出现 battle 的场面。在这个讨论过程中，甚至你不发言都是可以的，慢慢的你就放松下来，思路也会越来越清晰。本人就是那个全程没说一句话的围观群众，有人比我更惨的吗。本来当时我想到一个对策，做了好久的心理斗争，鼓起勇气准备发言，结果就撞上好几个人都想发言。我自然就：‘你先来，你先来’，错过了这个黄金机会。结果到讨论结束再也没有勇气在说话了，我就是这个群面中的哑巴。

    # Q&A

    最后一个部分就是 hr 的问答时间。因为我全程没说话我已经知道，不会问我问题了，我这个时候已经彻底放松下来了。开始竖起👂听 hr 会问什么问题，想验证一下我在面经看的一样不。

    • HR 首先提问了 timer。我们的讨论提前四分钟结束，问 timer 认为自己如何分配时间的，以及有什么可以优化的地方
    • 然后问了讨论中提出发言比较频繁的同学，具体问的什么问题我给搞忘了。😊
    • 最后提问了最开始破冰的那位女同学。问她对于这场讨论自己的定位是什么？讨论中一些同学发言较少出现的原因，以及如何改善这个问题？对于整场讨论 leader 应该履行什么样的职责？

    相反开始口若悬河的那位把自己放在 leader 位置男同学并没有被问到任何问题，具体原因不知道，但是我个人感觉可能是说的话虽然多，但是有效输出不多。相比破冰女一号，她说话就简洁自信，逻辑清晰，起到了推动讨论进程总结的作用，她才是那个隐藏 boss，人狠话不多型。

    # 为什么不做程序员？

    人作为一个社会动物，我觉得通过人与人之间的相处更容易找到意义。转了管理，考虑的不仅仅是自己，开始懂得照顾他人，关心团队的业绩，队员的成长。在对团队坚持不懈的支持下，团队取得的卓越绩效，队员的稳步成长，比独自撸代码战胜难题，更能带来满足感。

    上面说了我从程序员转型管理最重要的动机。然而我并不想盲目鼓励所有的程序员都去着急的转管理。这里有两个很重要的原因，首先，程序员和技术经理，不是黑与白的对立。管理特别是在技术领域下，需要程序员时期技术的沉淀，这样一来能更加深入的理解程序员这个群体的需求，二来帮助你得到程序员们新的信任，三是有助于敏锐的发现项目中暗藏的技术风险

    除了需要技术积累，选择管理并不是对所有的人都适合。如果你喜欢专注起来做一件事，喜欢钻研事物的本质而成为某个领域的专家，比如某个数据库，或者某个编程语言。做程序员最大的幸事，就是你既可以做自己喜欢的，也可以衣食无忧。

    为什么要转型？如果你还没有答案，也不要过于纠结，大不了试一试，喜欢了就接着做，不喜欢可以继续做回工程师。这其实不但可行，反而会有意外的收获，因为管理，可以使你成为更好的工程师。因为从事管理，可以锻炼你更加全面长远的看待问题。也可以让你接触到更多的问题及解决办法，进而增加处理问题的经验。凡是没有绝对的黑和白，不同的岗位有不同的优缺点，如何找到合适你的，这是一个值得整个职业生涯都去考虑的事
